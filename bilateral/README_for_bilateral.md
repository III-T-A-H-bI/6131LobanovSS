# Bilateral

Реализация билатерального фильтра для 8-битных grayscale изображений с использованием CUDA для ускорения на GPU. Проект включает CPU и GPU реализации для сравнения производительности и проверки корректности.


## 🎯 Описание

Билатеральный фильтр - это нелинейный фильтр, который сохраняет края изображения while удаляя шум. В отличие от обычного гауссовского размытия, билатеральный фильтр учитывает не только пространственное расстояние между пикселями, но и разницу в их интенсивности.

Этот проект демонстрирует:
- Реализацию билатерального фильтра на CPU (для валидации)
- Оптимизированную реализацию на GPU с использованием CUDA текстур
- Сравнение производительности CPU vs GPU
- Работу с BMP файлами (8-bit grayscale)

## ✨ Особенности

- 🚀 **Ускорение GPU**: Использование CUDA текстур для оптимального доступа к памяти
- 📊 **Сравнение производительности**: Автоматическое измерение времени выполнения на CPU и GPU
- ✅ **Валидация результатов**: Сравнение выходных изображений CPU и GPU
- 🖼️ **Поддержка BMP**: Встроенные функции для загрузки и сохранения 8-bit grayscale BMP
- ⚙️ **Настраиваемые параметры**: Возможность изменения sigma_d и sigma_r
- 📈 **Детальная статистика**: Вывод различий между CPU и GPU результатами

## Сборка и запуск

### 1. Генерация тестового изображения (опционально)

Скрипт `generate_test.py` создаёт изображение `test_input.bmp` размером 512×512 пикселей. Оно представляет собой линейный градиент от чёрного к белому с наложенным гауссовским шумом (σ=30). Это позволяет наглядно оценить сглаживающее действие билатерального фильтра.

```bash
python3 generate_test.py
```

### 2. Компиляция программы

Убедитесь, что архитектура вашей GPU указана правильно (в примере `sm_89` для RTX 4090). Проверить архитектуру можно командой `nvidia-smi --query-gpu=compute_cap --format=csv`.

```bash
nvcc -O3 -arch=sm_89 bilaterial.cu -o bilaterial
```

Для старых GPU (например, GTX 1080) используйте `-arch=sm_61`.

### 3. Запуск фильтрации

Синтаксис:
```bash
./bilaterial <input.bmp> <output_gpu.bmp> <output_cpu.bmp> [sigma_d] [sigma_r]
```

- `sigma_d` – пространственный параметр (стандартное отклонение в пикселях). По умолчанию 1.5.
- `sigma_r` – параметр яркости (стандартное отклонение в единицах яркости 0–255). По умолчанию 25.0.

Пример:
```bash
./bilaterial test_input.bmp result_gpu.bmp result_cpu.bmp 1.5 25.0
```

После выполнения программа выводит:
- Время работы CPU и GPU
- Ускорение (Speedup)
- Статистику различий между результатами (максимальная, средняя разница, процент пикселей с разницей >1)

### 4. Быстрый запуск через скрипт `bilaterial.sh`

Скрипт автоматически генерирует тестовое изображение, компилирует и запускает фильтрацию с параметрами по умолчанию.

```bash
chmod +x bilaterial.sh
./bilaterial.sh
```

## Описание алгоритма

Билатеральный фильтр заменяет значение каждого пикселя на взвешенное среднее соседних пикселей, где веса учитывают как **пространственную близость** (гауссово ядро), так и **близость по яркости** (гауссово ядро от разницы значений). Это позволяет сглаживать шум, сохраняя резкие границы.

### Формула для пикселя `(x,y)`:

```
I_filtered(x,y) = ( Σ I(x+dx, y+dy) * w_s(dx,dy) * w_r(|I(x+dx,y+dy) - I(x,y)|) ) / Σ w_s * w_r
```

где:
- `w_s(dx,dy) = exp(-(dx²+dy²) / (2σ_d²))`
- `w_r(δ) = exp(-δ² / (2σ_r²))`

### Особенности реализации

- **Размер окна**: 3×3 (для соответствия версии CPU, но может быть легко расширен).
- **Обработка границ**: в CPU – clamp (ближайший существующий пиксель); в GPU – используется режим адресации `cudaAddressModeClamp` (повтор крайних значений).
- **Использование текстур**: входное изображение загружается в текстуру с режимом чтения `cudaReadModeNormalizedFloat`, что автоматически преобразует значения `unsigned char` (0–255) в числа с плавающей точкой в диапазоне `[0.0, 1.0]`. Это упрощает вычисления и повышает точность.
- **Нормализация sigma_r**: поскольку GPU работает с нормализованными яркостями `[0.0,1.0]`, переданный параметр `sigma_r` (в диапазоне 0–255) делится на 255.0.
- **Ядро CUDA**: каждый поток обрабатывает один выходной пиксель. Используется текстурный объект для чтения соседних пикселей без дополнительной адресации.

## Выходные данные

- **result_gpu.bmp** – изображение после билатеральной фильтрации, вычисленное на GPU.
- **result_cpu.bmp** – изображение после фильтрации на CPU (эталон).
- В консоль выводится подробная статистика различий и время выполнения.

## Верификация корректности

- Для тестового изображения с шумом результаты CPU и GPU визуально неотличимы.
- Статистика различий обычно показывает среднюю разницу менее 0.5 и максимум до 2–3 единиц (что соответствует погрешности округления при преобразовании `float` → `unsigned char`).
- При одинаковых параметрах фильтра и одинаковой обработке границ алгоритмы дают идентичные результаты с точностью до округления (если используется одинаковая стратегия на краях).
